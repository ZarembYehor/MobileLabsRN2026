# Лабораторна робота №2: Побудова вкладеної навігації та оптимізація відображення великих списків у React Native із використанням компонентів FlatList та SectionList.

## Мета

ознайомлення з принципами навігації у мобільних застосунках React Native

## Опис реалізованого функціоналу

### 1. Списки та оптимізація

- **Стрічка новин:** Реалізовано динамічне завантаження даних з імітацією мережевих запитів.
  - Pull-to-Refresh: Оновлення списку свайпом зверху.
  - Infinite Scroll: Дозавантаження нових постів при досягненні кінця списку.
  - Компоненти: Використано кастомні Header, Footer та розділювачі.
  - Оптимізація: Налаштовано параметри initialNumToRender, maxToRenderPerBatch та windowSize для плавності роботи.

- **Екран контактів:** Групування контактів за алфавітом із використанням липких заголовків секцій.

### 2. Навігація

- **Вкладена навігація:** Використано структуру Drawer Navigator, всередині якого знаходиться Native Stack Navigator.
- **Динамічні заголовки:** Заголовок екрана деталей автоматично змінюється залежно від назви обраної новини.
- **Кастомне меню:** Створено власний компонент для Drawer, що містить інформацію про автора та зручні переходи.

### Чим відрізняється FlatList від ScrollView?
ScrollView рендерить усі свої дочірні компоненти одночасно одразу після завантаження. Це підходить для коротких сторінок. FlatList рендерить лише ті елементи, які зараз видно на екрані, що значно економить пам'ять та ресурси при роботі з великими масивами даних.

### Що таке віртуалізація списків?
Це механізм, який використовує FlatList для підтримки високої продуктивності. Він полягає у підтримці обмеженого вікна рендерингу та заміні елементів, що вийшли за межі екрана, на порожні заглушки, зберігаючи при цьому стабільну позицію прокрутки.

### Як здійснюється передача параметрів між екранами?
Параметри передаються як другий аргумент функції навігації. На цільовому екрані ці дані отримуються через об'єкт route.params.

### Що таке вкладена навігація?
Це ситуація, коли один навігатор рендериться всередині екрана іншого навігатора. Наприклад, у цій роботі Stack.Navigator є компонентом одного з екранів Drawer.Navigator. Це дозволяє комбінувати різні типи переходів.

### У яких випадках застосовується SectionList?
SectionList використовується, коли дані мають логічне групування або ієрархію. Найпоширеніші приклади: список контактів, розклад занять або меню ресторану.
